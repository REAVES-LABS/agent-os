A drop-in supervision layer for AI agents. Every action gets a verdict, every category accrues a trust score, every failure gets a recovery plan. Speaks MCP. LLM-agnostic.

npm install -g @reaves-labs/agent-os 30-second quickstart →
MCP server CLI SQLite, no service Anthropic · OpenAI · Ollama · any OAI-compatible MIT

If an agent is the user, software needs machine-native paths, permissions, and recovery.

YC Request for Startups #12. Most agent stacks today are a router + a model + a tool dispatcher. None of them have the supervision substrate that turns "the model said do X" into "do X only if the trust score for X is above the floor, and only if X is reversible." This is that substrate.

PATHS

Machine-native filesystem

Outbox for auto-approved writes. Approval queue for anything risky. Drop a file, get a verdict.

PERMISSIONS

Per-category trust scores

Capability × Quality × Impact = worth. Worth compounds into a trust score that gates the next verdict.

RECOVERY

Plans on failure

When an action fails, the supervisor returns a structured retry / rollback / escalate plan. Not just an error.

Four tools. That's it.

submit_action → verdict + trust score

Agent says "I want to do X." Returns auto / supervised / escalate. Auto goes straight to outbox; the rest goes to the human approval queue.

{
  "agentId":        "trader-7",
  "category":       "place_trade",
  "action":         "Open $200 BTC long at market",
  "irreversibility":"irreversible"
}
record_outcome → updated trust score

After execution, report the three axes. capability × quality × impact moves the rolling trust score for that category.

{
  "actionId":   "...",
  "success":    true,
  "capability": 1.0,
  "quality":    0.85,
  "impact":     0.7
}
get_routing → which sub-agent should own this

Hand a task description, get a single role back with a confidence score and one-sentence rationale. Use it to fan out.

recover → structured retry / rollback / escalate

Submit a failed action ID + error message. Get back a structured plan: a tighter prompt to retry with, a rollback step, or an escalation target.

The supervisor is not Claude. Or OpenAI. Or anyone.

A supervisor is a decision-maker, not a brand. agent-os ships with four backends out of the box and a generic adapter for everything OpenAI-compatible. Run it on free local Ollama or on a frontier API — the engine doesn't care.

Pick by env var

AGENT_OS_SUPERVISOR=ollama       # free, local
AGENT_OS_SUPERVISOR=anthropic    # frontier
AGENT_OS_SUPERVISOR=openai       # frontier
AGENT_OS_SUPERVISOR=generic      # any OAI-compatible
                                 # (Groq, Together, vLLM, llama.cpp)

Or pass one in

import { AgentOS } from "@reaves-labs/agent-os";

const os = new AgentOS({
  supervisor: { backend: "ollama", model: "llama3.2" }
});

30-second quickstart

  1. 1.

    Install

    npm install -g @reaves-labs/agent-os
  2. 2.

    Submit an action from the CLI

    agent-os submit send_email "Email Jane the Q3 deck" \
  --irreversibility=external

    Returns a verdict. Reversible action goes to ~/.agent-os/outbox/; risky goes to ~/.agent-os/queues/approval/.

  3. 3.

    Wire it as an MCP server

    Add to your agent client's MCP config (Claude Desktop, Cursor, OpenAI Codex, custom):

    {
  "mcpServers": {
    "agent-os": { "command": "agent-os-mcp" }
  }
}

Why this is different from "just use a guardrails library"

What guardrails libraries do

  • → Static rules per call
  • → Pass / fail at the moment of inference
  • → No memory across actions
  • → No notion of irreversibility
  • → No recovery plan when something breaks

What agent-os does

  • → Per-category trust scores that compound
  • → Three-axis verdict (auto / supervised / escalate)
  • → SQLite memory of every action and outcome
  • → Irreversibility floor: irreversible = always escalate
  • → Structured recovery plans, not raw errors
